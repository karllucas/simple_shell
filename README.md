<p dir="auto"><strong>Shellby</strong> is a simple UNIX command language interpreter that reads commands from either a file or standard input and executes them.</p>

<p dir="auto">Alternatively, if command line arguments are supplied upon invocation, <strong>shellby</strong> treats the first argument as a file from which to read commands. The supplied file should contain one command per line. <strong>Shellby</strong> runs each of the commands contained in the file in order before exiting.</p>

<p dir="auto">After receiving a command, <strong>shellby</strong> tokenizes it into words using <code>" "</code> as a delimiter. The first word is considered the command and all remaining words are considered arguments to that command. <strong>Shellby</strong> then proceeds with the following actions:</p>
<ol dir="auto">
<li>If the first character of the command is neither a slash (<code>\</code>) nor dot (<code>.</code>), the shell searches for it in the list of shell builtins. If there exists a builtin by that name, the builtin is invoked.</li>
<li>If the first character of the command is none of a slash (<code>\</code>), dot (<code>.</code>), nor builtin, <strong>shellby</strong> searches each element of the <strong>PATH</strong> environmental variable for a directory containing an executable file by that name.</li>
<li>If the first character of the command is a slash (<code>\</code>) or dot (<code>.</code>) or either of the above searches was successful, the shell executes the named program with any remaining given arguments in a separate execution environment.</li>
</ol>

<li><code>alias NAME='VALUE' [...]</code>: Defines an alias for each <code>NAME</code> whose <code>VALUE</code> is given. If <code>name</code> is already an alias, its value is replaced with <code>VALUE</code>.</li>
</ul>
